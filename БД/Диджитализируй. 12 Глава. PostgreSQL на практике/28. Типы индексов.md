Типы индексов
*  B-TREE (сбалансированное дерево). С ним работали во всех уроках до. Используется по дефолту. Может использоваться не только при равенстве, но и при условиях больше, меньше и так далее. Такие индексы могут использоваться не только при фильтрации с `where`, но и при сортировке. Так как в btree-индексе данные лежат в уже отсортированном виде, то этот результат сортировки удобно использовать вместо повторного выполнения сортировки на каждом запросе.
* HASH - используется при равенстве (например where = 'abacaba'). Работает так, что мы берём хеш ключа (от колонки которую индексируем) и значение этого ключа указывает на место в котором хранятся ссылки на строки таблицы.
* GIST - приближённый поиск, чаще всего по тексту (например поиск слова по тексту, в котором может быть опечатка). Также используется для географических данных.
* SP-GIST - используется для поиска по географическим данным, в которых есть сегментация данных.
* GIN - используется для полнотекстового поиска и для индекса сложных структур (вроде массива, jsonb и так далее).
* BRIN - эффективны для индексации огромных массивов данных, где приоритет отдаётся размеру индекса.
