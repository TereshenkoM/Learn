
Транзакции - атомарный набор операций
То  есть, либо все операции в наборе должны быть выполнены, либо ни должно быть выполнено ни одной.

Базовый пример - банковские операции. Например, user_1 должен перевести 100р user_2. Мы не можем допустить такого, чтобы у user_1 деньги списались, а user_2 при этом их не получил.

Пример транзакции
```postgresql
begin;

update bank_account set balance = balance - 1000 where user_id=1;
update bank_account set balance = balance + 1000 where user_id=2;

commit;
```

begin - старт транзакции.
commit - применить транзакцию.

Если же есть ошибка при выполнении операций, то транзакции откатываются.

```postgresql
begin;


update bank_account set balance = balance - 1000 where user_id=1;
update bank_account set balance = balance + 1000 where user_id=2;

rollback;
```

rollback - откат транзакции (отмена изменений в БД).

Также внутри транзакции пользователь позволяет менять структуру самих таблиц.
