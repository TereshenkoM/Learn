
Есть два основных типа обработки данных
* OLTP - Online Transaction Processing - обработка транзакций в реальном времени - это системы, которые выполняют короткие операции меняющие данные. Таким системам характерны быстрые транзакции которые обрабатывают небольшой объём данных и есть операции чтения и записи. Postgresql - OLTP система.

* OLAP - Online Analytics Processing - Обработка больших объёмов данных для задач аналитики, например логика в админке предназначенная для сотрудников сервиса. Характерны редки операции записи. В большинстве своём это отчёты по уже загруженным данным в систему (отчёты могут строиться дольше, чем выполняется транзакция я OLTP системе). Данные в OLAP не всегда актуальны, сую моментные данные.  Есть такой термин - ETL - extract, transform, load - извлечь, преобразовать и загрузить. Например логи. Мы доставли логи, обработали логи и загрузили в OLAP хранилище для обработки. Пример OLAP системы - GreenPlum, ClickHouse -  Данные там хранятся по колонкам в денормализованном виде.

По хорошему мы можем делать аналитику и в OLTP системе, но это может создать очень высокую нагрузку.
