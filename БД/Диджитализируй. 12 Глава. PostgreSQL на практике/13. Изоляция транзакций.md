Есть 4 классических уровня изоляции:
- **Read uncommitted**, то есть чтение незакомиченных данных    
- **Read committed**, то есть чтение закомиченных данных
- **Repeatable read**, то есть повторяемое чтение
- **Serializable**, то есть сериализуемость.

Позже подробнее о каждом уровне.

Они расположены он уровня с самым большим количеством аномалий (read uncommited), до уровня с самым маленьким количеством аномалий (serializable).

**Аномалия** - некорректные или неожиданные результаты которые мы можем видеть при конкурентном выполнении запросов.

Например: аномалия грязного чтения
```postgresql
-- first
update account set balance=1000 where account_id=1;
-- баланс был 1000 руб на начало траназкций
begin;
update account set balance=balance-900 where account_id=1;

--second
begin;
update account set balance=balance-100 where account_id=1;
-- запишется баланс 0 рублей в итоге
```

Вторая транзакция повиснет и будет ждать выполнения первой.
**При этом важно, что так происходит потому, что в Postgres нет Read uncommitted из за MVCC

По умолчанию Postgres всегда использует **Read committed**, однако мы можем указать уровень изоляции при открытии транзакции
```postgresql
begin transaction isolation level read committed;
```

**В других СУБД READ UNCOMMITTED может быть реализован так, как описано в стандарте SQL и там будет возникать грязное чтение.**
