Индексы могут быть использованы для проверки на уникальность.

Пример
Cоздадим таблицу и наполним данными

```postgresql
create temp table some_uniq_value (
	some_uniq_value_id bigint generated always as identity primary key,
	value varchar(30) unique
);

insert into some_uniq_value (value) values ('hello'), (''), ('bye'), (null);
```

Вставить строку hello и bye мы не можем, однако можем вставить null
```postgresql
insert into some_uniq_value (value) values
(null);
-- null можно вставить сколько угодно, потому что null=null это null
```

Для создания уникального индекса
```postgresql
create unique index unique_index_name on some_uniq_value(some_uniq_value_id);
```

При этом важно помнить, что помимо проверки уникальности этот индекс также может ускорять поиск (так как это всё также b-tree индекс). При этом **Вставки и обновления могут замедлиться из-за дополнительной проверки уникальности**
