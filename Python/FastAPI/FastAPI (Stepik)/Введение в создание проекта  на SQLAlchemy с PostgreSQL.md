## Установка драйверов для взаимодействия вашего Python-кода с базой данных.

Для работы нам потребуется синхронный и асинхронный движок (драйвер) работы с нашей базой данных, поэтому (если ещё не сделали этого) установите их командами:

```
pip install psycopg2
```

и

```
pip install asyncpg
```

Последний драйвер (вроде как по тестам, которые были в открытом доступе) быстрее, чем `**psycopg3**`. Для Linux'а вместо psycopg2 нужен **`psycopg2-binary`**.


## Структура проекта

```
my_project
├── app
│   ├── api
│   │   ├── endpoints
│   │   │   ├── __init__.py
│   │   │   └── todo.py  # Файл для маршрутов к неким todo
│   │   ├── schemas
│   │   │   ├── __init__.py
│   │   │   └── todo.py  # файл для pydantic-моделей, связанных с тудушками
│   │   └── __init__.py
│   ├── core
│   │   ├── __init__.py
│   │   └── config.py  # тут будет наш конфиг для приложения
│   ├── db
│   │   ├── __init__.py
│   │   ├── database.py  # тут будет настройка нашей БД
│   │   └── models.py  # тут будут лежать sqlalchemy-модели, которые отобразят                  (отразят) наши таблицы в БД
├── .gitignore
├── .env
├── README.md
├── requirements.txt
└── main.py
```

Так как теперь мы работаем с базой данных, то нам потребуется создавать **модели**, которые представляют из себя питонячье отображение на модели (таблицы) нашей БД (объектно-реляционное отображение). Чтобы не было конфликта имен, директорию с Pydantic-моделями принято переименовывать в `schemas`, как представлено в данной структуре. Иначе было бы две директории с моделями (когда проект разросся бы) - **models (pydantic) и models (sqlalchemy)** -> выход из этого конфликта обозначен.
